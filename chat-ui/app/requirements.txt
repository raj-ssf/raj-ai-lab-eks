# Chainlit 2.x — version history skipped 1.4/1.5 entirely (1.3.2 →
# 2.0.0 was the next stable). 2.x has Keycloak in the built-in OAuth
# provider list, registered automatically when OAUTH_KEYCLOAK_*
# env vars are set. 1.3.2 didn't and a runtime-appended custom
# provider hit a route-registration vs decorator-check mismatch
# (decorator's "any provider configured?" check passed, but HTTP
# routes were enumerated from a hardcoded list, not runtime
# providers — so /auth/oauth/keycloak/login fell through to SPA
# and the browser looped).
#
# 2.x has minor API drift from 1.3 (cl.Action / cl.Step / data
# layer); we pin to the major to absorb whatever breaks at runtime
# in one bound. Pin exactly after first successful build.
chainlit>=2.0.0,<3.0.0
