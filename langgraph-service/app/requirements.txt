# Langfuse v3 — matches the in-cluster server (3.43.0 at time of write).
# The v2 SDK posts to legacy ingestion endpoints that v3 server rejects,
# so traces silently drop. v3 SDK reorganized imports
# (langfuse.callback → langfuse.langchain) and switched to per-call
# metadata for user_id/tags rather than constructor kwargs. Pinned as a
# range until first successful GHA build, then `pip freeze` the resolved
# version and pin exactly. rag-service runs the same pin migration in a
# follow-up commit so both services emit into the v3 backend.
langfuse>=3.0.0,<4.0.0

# release in lock-step, so they have to move together — pinning any
